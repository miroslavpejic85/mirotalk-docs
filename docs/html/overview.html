MiroTalk SFU

Selective Forwarding Unit

Primary Use Case

Large meetings/classes and webinars

8+ participants (up to ~100 per single CPU)

Unlimited rooms

Security

DTLS/SRTP
Encrypted in transit using DTLS/SRTP protocols. Server (your) can access media streams.

Key Features

  • All P2P features plus:
  • Waiting/lock room
  • Hand raise
  • Polls
  • Virtual backgrounds
  • File transfer
  • Collaborative editor
  • RTMP stream
  • Webinar tools & roles

Performance

Highly scalable
Each user uploads one stream
Slightly higher latency
Server capacity limit

Compatibility

Browser-based
Needs decent resources
Up to 8K video

Deployment

Self-host SFU server
Docker or PM2

Requirements

Ubuntu 20.04/24.04
3 CPU Cores
